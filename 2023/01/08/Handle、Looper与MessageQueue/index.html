为什么使用Handle

Andriod不允许在子线程中更新界面组件，因为多个子线程并发操作UI可能导致性能、线程安全问题。
Android制定规则：只允许在UI线程（主线程）中修改Activity里的UI组件。
程序第一次启动时，会同时启动一条主线程，用于处理UI相关事件，如用户按键、用户触屏、屏幕绘图等，并把相关事件分发到对应的组件上进行处理。
那么如何让新启动的线程来动态改变界面组件的属性值？
Android使用Handle的消息传递机制来实现。

Handle类简介

Handle类作用：

  • 让主线程“适时”的处理新启动的线程所发送的消息。（回调实现）

上面说法需要分成两步：

  1. 新启动的线程发送消息。
  2. 主线程获取并处理消息。

这里显然通过回调的方法来实现，重写Handle类中处理消息的方法：

  1. 新启动的线程发送消息，发送到MessageQueue
  2. Handle不断从MessageQueue中获取并处理消息（这里一直在回调处理消息的方法）

Handle类主要方法如下：

  • handleMessage(Message msg)：处理消息的方法。
  • hasMessage(int what)：检查队列中是否包含属性值为what的消息。
  • hasMessage(int what, Object object)：检查队列中是否包含属性值为what且object属性为指定对象的消息。
  • Message obtainMessage()：获取消息（多个重载）。
  • sendMessage(Message msg):立即发送消息。
  • sendMessageDelayed(Message msg, long delayMillis):指定毫秒后发送消息。

借助于上述方法，程序可以很方便的借助于Handle来进行消息传递
